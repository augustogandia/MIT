### 2. Proof of Existence (Cryptographic Verification)
To protect against plagiarism and claims of earlier publication, this project utilizes the following verification methods:

*   **GitHub Verified Commits:** Every update to this text is made via the GitHub Web Editor, which applies a cryptographic signature. This provides a server-side `verified_at` timestamp that serves as third-party evidence of the content's existence on a specific date.
*   **Immutable History:** This repository uses a linear history. Any attempt to alter past publication dates would invalidate the cryptographic signatures.

### 3. Blockchain Timestamping (TSA)
*   **Provider:** [OpenTimestamps](opentimestamps.org)
*   **Status:** Original files are hashed and anchored to the Bitcoin blockchain.
*   **Verification:** To verify the timestamp of a file, upload the `.ots` file found in this repository along with the original document to the OpenTimestamps main site.
